Create a monitor

Fill in the spec for a new monitor — Tax, ESG, employment, M&A antitrust, or your own scope. On submit you'll get (1) a downloadable monitor-blueprint.md and (2) a Claude-ready prompt that scaffolds the monitor folder, the agent cell, the website page, and starts the first calibration run.

1 · Identity

What this monitor is called.

2 · Topics

These become the topic tabs on the monitor page. 2–6 works best.

3 · Anchor

Who is this monitor calibrated for? Drives ranking and feedback.
Single-named = one human's mental model. Operator-anchored = K. Maleevska runs it. Multi-persona = several consumers share one feed.

4 · Scope

Jurisdictions in scope, and Prosus portfolio companies relevant.

5 · Sources

One source per line. Tier A = regulators + courts. Tier B = trade press + firm alerts. Tier C = general business news.

6 · Filters & ranking rules

Locked filters apply on top of the system-wide 1–8. Auto-rank rules force items to Rank A regardless of generic scoring.

7 · Review & ship

Two artefacts: a Markdown blueprint and a Claude prompt.
What happens next. Paste the Claude prompt into a Claude Code session at the repository root. Claude will scaffold monitors/monitor-XX-<slug>/, the 5+1 agent cell, the .claude/agents/mXX-* subagent wrappers, the website pages under website/monitor-XX/, and update CLAUDE.md, strategy.md, and the hub card. Then it will run the first browser-intake pass and produce a calibration handoff brief in generated/handoffs/.
